⚙️ Customization (maden-config.css)

maden-config.css is the core file that centrally defines all the fundamental building blocks of MadenCSS. Everything, from the color palette and spacing system to font sizes, borders, and shadows, is defined as **CSS Variables** within the :root block. By editing this single file, you can instantly adapt the entire system to your project's design language.

1. Color Palette and Theming (Dark / Light Mode)

All color utility classes (e.g., bg-primary, text-danger) utilize these variables. For ensuring high contrast, the corresponding text color for each background is also defined using the --on-colorname format.

Theme Support:

The default values (`:root` block) hold the **Dark Theme** colors. Light theme colors are defined inside the html[data-theme="light"] block, allowing for easy, zero-JavaScript theme switching. 


:root{
  /* Surfaces (Default: Dark Theme) */
  --background:   #0b1220;
  --on-background:#ffffff;

  --surface:      #0f172a;
  --on-surface:   #e0e0e0;  
  
  /* Semantic Colors */
  --primary:      #f0bb0c; 
  --on-primary:   #000000;

  --success: #4caf50;
  --on-success: #000;             
  
  --danger:    #dc3545;
  --on-danger: #000;    
  
  /* ... Other variables ... */
}

/* Light Theme Configuration Example */
html[data-theme="light"] {
  --background: #ffffff;
  --on-background: #141414;
  /* ... Continued ... */
}

**Customization:** To change the main accent color in the Dark theme, simply modify the --primary variable within the :root block.

2. Spacing System and Max Widths

MadenCSS's entire spacing system is built on a 4-pixel base unit, defined by --spacing-unit.

These variables determine the values for all margin, padding, and gap utility classes, such as m-1 (4px), p-3 (16px), and grid-gap-2 (8px). 


:root{
  /* Spacing (4px System) */
  --spacing-unit: 4px;
  --spacing-1: calc(var(--spacing-unit) * 1);   /* 4px  -> Utility: p-1, m-1 */
  --spacing-3: calc(var(--spacing-unit) * 4);   /* 16px -> Utility: p-3, m-3 */

  /* Maximum Widths (Layout) */
  --max-width-lg: 1200px; /* Utility: max-width-lg, used with page-center. */
  --max-width-sm: 767px; 
}

Scaling: Changing the --spacing-unit variable allows you to rescale all spacing and gap values across the entire system from a single location.

3. Typography (Font Weights, Sizes, and Families)

Font weights (fw-thin to fw-black) and font families are defined here.

Font sizes (fs-sm to fs-huge) are defined using the modern **CSS clamp()** function. This ensures that text scales fluidly and responsively based on the viewport size, providing superior readability across all devices. 


  /* Font Weights */
  --font-weight-light: 200;
  --font-weight-normal: 400;
  --font-weight-bold: 700;
  --font-weight-black: 900; /* Utility: fw-black */
  

  /* Font Families */
  --font-family-base: system-ui, -apple-system, ... sans-serif;
  --font-family-mono: 'SFMono-Regular', Consolas, ... monospace; /* For code blocks */


  /* .fs-sm    { font-size: clamp(var(--fs-sm-min), var(--fs-sm-fluid), var(--fs-sm-max)); } */
  /* Font Sizes (fs-xxl example) */
  --fs-xxl-min: 1.5rem;      
  --fs-xxl-fluid: 2vw;       
  --fs-xxl-max: 2.25rem;     /* Scales smoothly between these two limits based on viewport width. */

4. Borders, Radius, and Shadow Configuration

MadenCSS also provides semantic variables for customizing border styles, corner rounding, and box shadows. 


  /* ---------- Radius / Border-Radius ---------- */
  --radius-xsm: 2px;
  --radius-sm:  4px;
  --radius:     6px;  /* Utility: radius (Base) */
  --radius-lg:  12px;
  --radius-full: 9999px; /* Utility: radius-full (Pill shape) */

  /* ---------- Border Widths ---------- */
  --border-thin: 1px;
  --border-normal: 2px; 
  --border-thick: 4px;  /* Utility: border-thick */

  /* ---------- Box Shadows ---------- */
  --shadow-light: 0 10px 30px rgba(0,0,0,.25); /* Utility: shadow-light */
  --shadow-normal: 0 10px 30px rgba(0,0,0,.40);
  --shadow-strong: 0 10px 30px rgba(0,0,0,.60); /* Utility: shadow-strong */

**Usage:** If you need all corner radii in your project to be softer/sharper, you can globally adjust these variables. For instance, increasing the value of --radius will increase all default rounding across the system.
