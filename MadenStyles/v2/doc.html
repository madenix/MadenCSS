Get Started With MADENCSS

MadenCSS is built on a few core concepts designed to maximize development speed, maintainability, and code predictability.

1. Create a basic html file.

Make sure <meta name="viewport"> included for responsive behaviour in mobile devices.


<!doctype html>
<html lang="en" data-theme="dark">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>MadenCss demo</title>
  </head>
  <body>
    <h1>Hello, world!</h1>
  </body>
</html>

2. Include MadenCSS base files.

Make sure maden-config.css (for configuration) and core.css (core utulities) are included the file between head tags.

<!doctype html>
<html lang="en" data-theme="dark">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>MadenCss demo</title>
    
    <!-- Link required CSS files -->
    <link rel="stylesheet" href="maden-config.css">
    <link rel="stylesheet" href="core.css">
    
  </head>
  <body>
    <h1>Hello, world!</h1>
  </body>
</html>

3. Open html file.

Open the html file in your browser of choice to see your Hello World! page. Congratulations. You have a MadenCss styled page already.

Now we can move further and see what else can be done!

1. Add following classes to the body tag
bg-background for background color, text-on-background to get contrast text on background, max-width-lg to limit the width of the body, page-center to center the page to the window. 

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>MadenCss demo</title>
    <!-- Link required CSS files -->
    <link rel="stylesheet" href="maden-config.css">
    <link rel="stylesheet" href="core.css">
    
  </head>
  <body class="bg-background text-on-background max-width-lg page-center">
    <h1>Hello, world!</h1>
  </body>
</html>

Refresh the page and get immediate result. But lets make our page more useful. We can get rid of the boring hello world and add a basic navbar.

2. MadenCss uses 12 column grid system. For the speed and simplicity MadenCss uses only 1 device breakpoint at 599px to switching the page for the mobile view. This value can be changed with hard coded ways but that is not recommended.

Copy and paste following code to your html file and see the result.

    <!-- Basic Navbar -->
    
  </body>
</html>

As you see we have created a basic navbar with ease and flexible atomic classes. Although you dont need to write components blocks, you can use pre-made MadenCss components.

CORE PRINCIPLES

MadenCSS is built on a few core concepts designed to maximize development speed, maintainability, and code predictability.

1. Utility-First & Component-Driven Speed

MadenCSS operates primarily on a **Utility-First approach**, where you compose designs using single-purpose classes (e.g., p-4, flex). However, to maximize design speed, the framework also provides essential, **ready-made components** like Modals, Sidebars, and Navbars.

This dual approach allows for ultimate flexibility through utilities while providing immediate solutions for common UI patterns. Crucially, these components are designed with **minimal (mostly zero) JavaScript dependency**, aligning with our commitment to simplicity. 

<!-- Compose unique designs using utilities -->
<div class="p-5 bg-surface radius shadow-light flex flex-col align-center">...</div>

<!-- Speed up workflow with built-in, zero-JS components -->
<label for="modal-state" class="btn bg-primary">Open Modal</label>

**Principle:** Utility for customizability, Components for velocity.

2. Semantic and Atomic Naming & Mobile-First Grid

MadenCSS prioritizes **predictability and maintainability** through highly semantic class names. Every atomic utility class directly reflects the single CSS property it applies, ensuring developers know exactly what the code does just by reading the class list. 

<!-- Semantic Utility Examples -->
<h1 class="fs-xxl fw-bold text-on-surface mb-4">...</h1>

<!-- Mobile-First Grid Integration (using distinct classes) -->
<div class="grid">
  <div class="col-4">Item 1</div>
  <div class="col-3">Item 2</div>
  <div class="col-5">Item 2</div>
</div>

**Benefit:** Consistency across the entire codebase and unparalleled ease of theming.

INSTALLATION

Start using MadenCSS with a simple local setup or CDN option.

All core utilities are included, making it easy to build responsive layouts instantly.

1. Local Setup (Recommended)

Download the required CSS files and place them in your project. Then include them inside your <head> tag. 

<!-- Link required CSS files -->
<link rel="stylesheet" href="maden-config.css">
<link rel="stylesheet" href="core.css">

Note: Import only the components you need. core.css is required for grid and base utilities.

2. CDN Usage (Optional)

If you prefer not to use local files, link the CDN version. 

<link rel="stylesheet" href="https://cdn.madencss.com/[version]/maden-config.css">
<link rel="stylesheet" href="https://cdn.madencss.com/[version]/core.css">
