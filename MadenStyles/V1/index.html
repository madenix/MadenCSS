INSTALLATION

Start using MadenCSS with a simple local setup or CDN option.

All core utilities are included, making it easy to build responsive layouts instantly.

1. Local Setup (Recommended)

Download the required CSS files and place them in your project. Then include them inside your <head> tag. 

<!-- Link required CSS files -->
<link rel="stylesheet" href="core.css">
<link rel="stylesheet" href="color.css">
<link rel="stylesheet" href="spacing.css">
<link rel="stylesheet" href="navbar.css">
<link rel="stylesheet" href="typography.css">
<link rel="stylesheet" href="sidebar.css">

Note: Import only the components you need. core.css is required for grid and base utilities.

2. CDN Usage (Optional)

If you prefer not to use local files, link the CDN version. 

<link rel="stylesheet" href="https://cdn.madencss.com/[version]/core.css">
<link rel="stylesheet" href="https://cdn.madencss.com/[version]/color.css">
<link rel="stylesheet" href="https://cdn.madencss.com/[version]/spacing.css">
<link rel="stylesheet" href="https://cdn.madencss.com/[version]/navbar.css">
<link rel="stylesheet" href="https://cdn.madencss.com/[version]/typography.css">
<link rel="stylesheet" href="https://cdn.madencss.com/[version]/sidebar.css">

3. Basic HTML Structure

After linking the files, you can start building layouts using the MadenCSS grid system. 

<body class="bg-background text-on-background">
  <div class="grid max-width-1200 page-center gap-4">
    <div class="col-3"> <!-- Sidebar --> </div>
    <div class="col-9"> <!-- Main Content --> </div>
  </div>
</body>

CORE PRINCIPLES

MadenCSS is built on a few core concepts designed to maximize development speed, maintainability, and code predictability.

1. Utility-First Approach

MadenCSS provides **single-purpose utility classes** (e.g., `p-4`, `flex`, `fw-bold`) that apply a single CSS rule. Instead of writing custom CSS, you build complex interfaces by **composing these classes directly in your HTML**. 

<!-- Traditional approach -->
<div class="card">...</div>

<!-- MadenCSS Utility-First -->
<div class="p-5 bg-surface radius shadow-light flex flex-col flex-center">...</div>

**Benefit:** Faster development, consistent design, and easier maintenance since styles are localized.

2. Mobile-First & Responsive Design

All MadenCSS utilities are **mobile-first** by default. Base classes (e.g., `col-6`) apply to all screen sizes. You use responsive prefixes (`sm:`, `md:`, `lg:`) only when you need to change behavior at a specific breakpoint. 

<!-- col-12 is default (mobile). lg:col-4 is applied on large screens. -->
<div class="grid">
  <div class="col-12 lg:col-4">...</div>
  <div class="col-12 lg:col-8">...</div>
</div>

MadenCSS uses a simple 12-column grid system built to adapt flawlessly across devices.

3. Semantic Color System

Colors are named based on their **purpose**, not their hue (e.g., `primary`, `success`, `danger`). This allows easy theme switching (Dark/Light Mode) and consistent feedback. All colors also have **`on-`** variants (e.g., `bg-primary` paired with `text-on-primary`) to ensure text readability automatically. 

<button class="bg-primary text-on-primary">Main Action</button>
<span class="bg-success-light text-success-dark">Order Shipped</span>

This system ensures accessibility and supports dynamic theme changes seamlessly.

COLORS

MadenCSS uses a **semantic color system** based on CSS variables, which enables seamless Dark and Light theme switching. All colors are defined by their **purpose** (`primary`, `success`, `danger`), ensuring consistent meaning across your application.

1. Semantic Color Palette

Every semantic color comes with a corresponding **`on-`** color (e.g., `text-on-primary`) to guarantee accessible contrast for text placed on that background.

Primary

Main brand color.

Success

Positive actions/feedback.

Danger

Critical actions/errors.

Warning

Cautions/alerts.

Info

Informational messages.

Surface

Component backgrounds.

2. Color Utility Classes

MadenCSS provides classes to apply colors to backgrounds (`bg-*`), text (`text-*`), and borders (`border-*`).

Example: Background & Text Colors

<!-- Primary button example -->
<button class="p-3 radius bg-primary text-on-primary fw-medium">
    Primary Action
</button>

<!-- Success status message -->
<div class="p-2 radius bg-success text-on-success">
    Order successfully processed.
</div>

Example: Border Color

<div class="p-4 border-2 border-full border-danger text-danger radius">
    Warning: This field is required.
</div>

**Note:** The border classes (`border-primary`, `border-danger`, etc.) are defined in `color.css`.

3. Neutral (Gray) Colors

For neutral interface elements, three shades of gray are provided: `dark`, `mid`, and `light`.

Dark

Used for primary text/icons (Dark Theme).

Mid

Secondary text/borders.

Light

Disabled elements/separators.
