CORE PRINCIPLES

MadenCSS is built on a few core concepts designed to maximize development speed, maintainability, code quality and predictability.

1. Utility-First & Component-Driven Speed

MadenCSS operates primarily on a **Utility-First approach**, where you compose designs using single-purpose classes (e.g., p-4, flex). However, to maximize design speed, the framework side also provides essential, **ready-made components** like Modals, Sidebars, and Navbars.

This dual approach allows for ultimate flexibility through utilities while providing immediate solutions for common UI patterns. Crucially, these components are designed with **minimal (mostly zero) JavaScript dependency**, aligning with our commitment to simplicity. 

<!-- Compose unique designs using utilities -->
<div class="p-5 bg-surface radius shadow-light flex flex-col align-center">...</div>

<!-- Speed up workflow with built-in, zero-JS components -->
<label for="modal-state" class="btn bg-primary">Open Modal</label>

**Principle:** Utility for customizability, Components for velocity.

2. Semantic and Atomic Naming & Mobile-First Grid

MadenCSS prioritizes **predictability and maintainability** through highly semantic class names. Every atomic utility class directly reflects the single CSS property it applies, ensuring developers know exactly what the code does just by reading the class list. This establishes a natural standard for both coding and collaboration. 

<!-- Semantic Utility Examples -->
<h1 class="fs-xxl fw-bold text-on-surface mb-4">...</h1>

<!-- Mobile-First Grid Integration (using distinct classes) -->
<div class="grid">
  <div class="col-4">Item 1</div>
  <div class="col-3">Item 2</div>
  <div class="col-5">Item 2</div>
</div>

**Principle:** Consistency across the entire codebase and unparalleled ease of theming.

3. Low Dependency

MadenCSS keeps dependencies to a minimum, because relying on too many makes adoption harder and slows things down. You can simply copy the source files into your project or use the CDN links—nothing more needed.

**Principle:** Simplicity and minimal dependency.

4. Optional Components

MadenCSS allows you to include only the components you need. Nothing is forced or bundled unnecessarily. If a component doesn’t serve your project, you can simply leave it out—keeping your codebase clean, lightweight, and free from bloat. 

<!doctype html>
<html lang="en" data-theme="dark">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>MadenCss demo</title>
    
    <!-- Link required CSS files -->
    <link rel="stylesheet" href="maden-config.css">
    <link rel="stylesheet" href="core.css">

    <!-- Link only the components you need -->
    <link rel="stylesheet" href="modal.css">
    
  </head>
  <body>
    <h1>Hello, world!</h1>
  </body>
</html>

Principle: Selective usage over unnecessary bulk.
