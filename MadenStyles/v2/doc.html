Installation

MadenCSS can be added to your project in two simple ways: using the CDN or including the source files locally. Choose whichever fits your workflow‚Äîno tooling or setup required.

1. CDN

The fastest way to get started. Add the stylesheet directly to your HTML: 

<link rel="stylesheet" href="https://cdn.madencss.com/maden.css">

2. Local Setup

Download the framework and copy the files into your project. Then link the CSS: 

<link rel="stylesheet" href="/css/maden.css">

CORE PRINCIPLES

MadenCSS is built on a few core concepts designed to maximize development speed, maintainability, code quality and predictability.

1. Utility-First & Component-Driven Speed

MadenCSS operates primarily on a **Utility-First approach**, where you compose designs using single-purpose classes (e.g., p-4, flex). However, to maximize design speed, the framework also provides essential, **ready-made components** like Modals, Sidebars, and Navbars.

This dual approach allows for ultimate flexibility through utilities while providing immediate solutions for common UI patterns. Crucially, these components are designed with **minimal (mostly zero) JavaScript dependency**, aligning with our commitment to simplicity. 

<!-- Compose unique designs using utilities -->
<div class="p-5 bg-surface radius shadow-light flex flex-col align-center">...</div>

<!-- Speed up workflow with built-in, zero-JS components -->
<label for="modal-state" class="btn bg-primary">Open Modal</label>

**Principle:** Utility for customizability, Components for velocity.

2. Semantic and Atomic Naming & Mobile-First Grid

MadenCSS prioritizes **predictability and maintainability** through highly semantic class names. Every atomic utility class directly reflects the single CSS property it applies, ensuring developers know exactly what the code does just by reading the class list. This establishes a natural standard for both coding and collaboration. 

<!-- Semantic Utility Examples -->
<h1 class="fs-xxl fw-bold text-on-surface mb-4">...</h1>

<!-- Mobile-First Grid Integration (using distinct classes) -->
<div class="grid">
  <div class="col-4">Item 1</div>
  <div class="col-3">Item 2</div>
  <div class="col-5">Item 2</div>
</div>

**Principle:** Consistency across the entire codebase and unparalleled ease of theming.

3. Low Dependency

MadenCSS keeps dependencies to a minimum, because relying on too many makes adoption harder and slows things down. You can simply copy the source files into your project or use the CDN links‚Äînothing more needed.

**Principle:** Simplicity and minimal dependency.

4. Optional Components

MadenCSS allows you to include only the components you need. Nothing is forced or bundled unnecessarily. If a component doesn‚Äôt serve your project, you can simply leave it out‚Äîkeeping your codebase clean, lightweight, and free from bloat. 

<!doctype html>
<html lang="en" data-theme="dark">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>MadenCss demo</title>
    
    <!-- Link required CSS files -->
    <link rel="stylesheet" href="maden-config.css">
    <link rel="stylesheet" href="core.css">

    <!-- Link only the components you need -->
    <link rel="stylesheet" href="modal.css">
    
  </head>
  <body>
    <h1>Hello, world!</h1>
  </body>
</html>

Principle: Selective usage over unnecessary bulk.

UTILITIES

‚öôÔ∏è Customization (maden-config.css)

maden-config.css is the core file that centrally defines all the fundamental building blocks of MadenCSS. Everything, from the color palette and spacing system to font sizes, borders, and shadows, is defined as **CSS Variables** within the :root block. By editing this single file, you can instantly adapt the entire framework to your project's design language.

1. Color Palette and Theming (Dark / Light Mode)

All color utility classes (e.g., bg-primary, text-danger) utilize these variables. For ensuring high contrast, the corresponding text color for each background is also defined using the --on-colorname format.

Theme Support:

The default values (`:root` block) hold the **Dark Theme** colors. Light theme colors are defined inside the html[data-theme="light"] block, allowing for easy, zero-JavaScript theme switching. 


:root{
  /* Surfaces (Default: Dark Theme) */
  --background:   #0b1220;
  --on-background:#ffffff;

  --surface:      #0f172a;
  --on-surface:   #e0e0e0;  
  
  /* Semantic Colors */
  --primary:      #f0bb0c; 
  --on-primary:   #000000;

  --success: #4caf50;
  --on-success: #000;             
  
  --danger:    #dc3545;
  --on-danger: #000;    
  
  /* ... Other variables ... */
}

/* Light Theme Configuration Example */
html[data-theme="light"] {
  --background: #ffffff;
  --on-background: #141414;
  /* ... Continued ... */
}

**Customization:** To change the main accent color in the Dark theme, simply modify the --primary variable within the :root block.

2. Spacing System and Max Widths

MadenCSS's entire spacing system is built on a 4-pixel base unit, defined by --spacing-unit.

These variables determine the values for all margin, padding, and gap utility classes, such as m-1 (4px), p-3 (16px), and grid-gap-2 (8px). 


:root{
  /* Spacing (4px System) */
  --spacing-unit: 4px;
  --spacing-1: calc(var(--spacing-unit) * 1);   /* 4px  -> Utility: p-1, m-1 */
  --spacing-3: calc(var(--spacing-unit) * 4);   /* 16px -> Utility: p-3, m-3 */

  /* Maximum Widths (Layout) */
  --max-width-lg: 1200px; /* Utility: max-width-lg, used with page-center. */
  --max-width-sm: 767px; 
}

Scaling: Changing the --spacing-unit variable allows you to rescale all spacing and gap values across the entire framework from a single location.

3. Typography (Font Weights, Sizes, and Families)

Font weights (fw-thin to fw-black) and font families are defined here.

Font sizes (fs-sm to fs-huge) are defined using the modern **CSS clamp()** function. This ensures that text scales fluidly and responsively based on the viewport size, providing superior readability across all devices. 


  /* Font Weights */
  --font-weight-light: 200;
  --font-weight-normal: 400;
  --font-weight-bold: 700;
  --font-weight-black: 900; /* Utility: fw-black */
  

  /* Font Families */
  --font-family-base: system-ui, -apple-system, ... sans-serif;
  --font-family-mono: 'SFMono-Regular', Consolas, ... monospace; /* For code blocks */


  /* .fs-sm    { font-size: clamp(var(--fs-sm-min), var(--fs-sm-fluid), var(--fs-sm-max)); } */
  /* Font Sizes (fs-xxl example) */
  --fs-xxl-min: 1.5rem;      
  --fs-xxl-fluid: 2vw;       
  --fs-xxl-max: 2.25rem;     /* Scales smoothly between these two limits based on viewport width. */

4. Borders, Radius, and Shadow Configuration

MadenCSS also provides semantic variables for customizing border styles, corner rounding, and box shadows. 


  /* ---------- Radius / Border-Radius ---------- */
  --radius-xsm: 2px;
  --radius-sm:  4px;
  --radius:     6px;  /* Utility: radius (Base) */
  --radius-lg:  12px;
  --radius-full: 9999px; /* Utility: radius-full (Pill shape) */

  /* ---------- Border Widths ---------- */
  --border-thin: 1px;
  --border-normal: 2px; 
  --border-thick: 4px;  /* Utility: border-thick */

  /* ---------- Box Shadows ---------- */
  --shadow-light: 0 10px 30px rgba(0,0,0,.25); /* Utility: shadow-light */
  --shadow-normal: 0 10px 30px rgba(0,0,0,.40);
  --shadow-strong: 0 10px 30px rgba(0,0,0,.60); /* Utility: shadow-strong */

**Usage:** If you need all corner radii in your project to be softer/sharper, you can globally adjust these variables. For instance, increasing the value of --radius will increase all default rounding across the framework.

üåô Theme Structure (Dark/Light Mode)

MadenCSS utilizes CSS Variables and the HTML data-theme attribute to enable fast and fluid switching between Dark and Light themes. This approach provides the flexibility to manage the theme with minimal JavaScript and easily supports native browser theme preferences.

1. Theme Mechanism

All color variables are defined in the maden-config.css file within two main blocks:

  • :root **Block:** Defines the color variables for the **Dark Mode** (the default theme of MadenCSS).
  • html[data-theme="light"] **Block:** Contains the **Light Mode** colors which override the Dark Mode variables when the data-theme attribute is set to "light".

By simply toggling the attribute on the html tag, you can instantaneously update the colors of the entire interface. No additional CSS files are needed for theme switching.

2. HTML Structure

You can set the default theme of your page using the data-theme attribute on the <html> tag:

Dark Mode


<html lang="en" data-theme="dark"> 
    <body class="bg-background text-on-background">...</body>
</html>

Light Mode


<html lang="en" data-theme="light"> 
    <body class="bg-background text-on-background">...</body>
</html>

3. Theme Toggling (Optional JavaScript)

To switch the theme based on user interaction, we recommend using a minimal JavaScript snippet to toggle the data-theme attribute between Dark and Light: 


function toggleTheme() {
    const htmlElement = document.documentElement;
    const currentTheme = htmlElement.getAttribute('data-theme');
    const newTheme = (currentTheme === 'dark') ? 'light' : 'dark';

    htmlElement.setAttribute('data-theme', newTheme);
    
    // Optional: Use localStorage to remember the user's preference
    localStorage.setItem('theme', newTheme);
}

// <button onclick="toggleTheme()">Toggle Theme</button>
‚ú® Tip: On page load, you can check the user's preference (from localStorage if available) or the browser's system preference (using the prefers-color-scheme media query) to set the initial theme.

3. Responsive Behavior (Breakpoint: 599px)

The grid system automatically applies an **auto-stacking** rule on all screens **599px and below**, ensuring mobile-friendly layouts by default.

No breakpoint-specific column classes are required.

  • Large Screens (> 599px): Columns preserve their defined .col-* widths.
  • Mobile Screens (‚â§ 599px): The grid switches to a single-column layout, and all columns expand to full width (100%).

Disabling Mobile Stacking (Opt-out)

If you want a grid to retain its 12-column structure even on mobile devices, add the .not-responsive class to the .grid container: 


<!-- This grid remains 4 / 8 columns even on mobile -->
<div class="grid grid-gap-2 not-responsive">
    <div class="col-4">...</div> 
    <div class="col-8">...</div> 
</div>

<h1 class="fs-huge">Hero title</h1>

2. How clamp() Works

The clamp() function defines a range in which the font size can grow or shrink:

  • Minimum: Small screens never go below this size
  • Fluid value: Scales based on viewport width
  • Maximum: Large screens never exceed this size

/* maden-config.css */
/* fs-base: clamp(0.875rem, 1vw, 1rem) */
--fs-base-min: 0.875rem;
--fs-base-fluid: 1vw;
--fs-base-max: 1rem;

/* core.css */
.fs-base {
  .fs-base  { font-size: clamp(var(--fs-base-min), var(--fs-base-fluid), var(--fs-base-max)); }
}
üí° Why this matters: No media queries, no breakpoints, no typography jumps ‚Äî just smooth, readable scaling across all devices.
